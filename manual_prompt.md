## ▼ ここからプロンプト ▼

あなたはドキュメント作成の専門家です。
以下に示すアプリの仕様情報をもとに、**エンドユーザー向けの操作マニュアル（PDF用）**を日本語で作成してください。

### 出力形式の指示

- PDF印刷を前提としたHTML形式で出力してください
- A4縦を想定したレイアウト（最大幅 900px 程度）
- 日本語フォント（Hiragino / Meiryo）
- 章ごとに番号付き見出しを使用
- 表・番号付き手順・注意ボックスを適切に使用
- 表紙・目次・各章・FAQ・フッターを含めること
- 印刷時に余白（margin: 20mm 程度）が確保されるようにすること
- `@media print` でナビゲーション要素を非表示にすること

### 全機能一覧と詳細仕様

#### 【1】アカウント・認証

**新規登録**
- メールアドレス＋パスワード（6文字以上）で登録
- 登録後に組織設定画面へ遷移

**ログイン**
- メール＋パスワードでログイン
- パスワードリセット機能あり（メール送信）

**ログアウト**
- メニュー（右上 ⋮）→「ログアウト」

**パスワード変更**
- メニュー→「パスワード変更」
- 現在のパスワードで再認証後、新パスワード（6文字以上）を設定

**ニックネーム変更**
- メニュー→「ニックネーム変更」
- ニックネームは履歴の記録者として使用される

**アカウント削除**
- メニュー→「アカウント削除」
- パスワード入力で本人確認後、完全削除（取り消し不可）

---

#### 【2】初回セットアップ（組織設定）

ログイン後に組織が未設定の場合、組織設定画面が表示される。

**A. 新しい組織を作成（管理者になる）**
1. 「新しい組織を作成」を選択
2. 組織名・組織コード（英小文字・数字・アンダースコアのみ）・ニックネームを入力
3. 利用規約・プライバシーポリシーに同意
4. 「作成」→ 承認待ち画面へ遷移（統括管理者による承認が必要）
5. 承認後に利用開始

**B. 既存の組織に参加（メンバーになる）**
1. 「既存の組織に参加」を選択
2. 管理者から共有された招待コードを入力
3. ニックネームを入力
4. 利用規約・プライバシーポリシーに同意
5. 「参加」→ 即時利用開始

**制限**
- ユーザー数の上限（デフォルト5人）に達している場合は参加不可
- 店舗数の上限（デフォルト5店舗）に達している場合は店舗追加不可

---

#### 【3】店舗一覧（メイン画面）

ログイン後の最初の画面。

**表示内容**
- 上部: 組織名
- 「店舗数: ○ 件」カード
- 各店舗カード（タップで店舗別在庫ページへ）
- 下部: 広告バナー（設定時のみ・1分ごとにローテーション）

**管理者による店舗追加**
- 右下の「＋」ボタン→ 店舗名・コード・店舗ID（英数字）を入力して追加
- 上限（デフォルト5店舗）に達するとエラー表示

**メニュー一覧（右上 ⋮）**
- 全店舗在庫確認（全員）
- 修正・追加履歴（全員）
- 商品マスタ管理（全員）
- 発注リスト（全員）
- 特別発注・新規発注（全員）
- 店舗の並び替え（全員）
- 組織管理（管理者のみ）
- 広告スペース管理（管理者のみ）
- ニックネーム変更（全員）
- パスワード変更（全員）
- 組織を脱退（メンバーのみ）
- 利用規約（全員）
- プライバシーポリシー（全員）
- 機能追加依頼 / 修正依頼 / バグ報告（全員）
- ログアウト（全員）
- アカウント削除（全員）
- バージョン表示（v1.2.0）

---

#### 【4】店舗別在庫管理

店舗カードをタップして開く。

**タブ**
- 商品 / テスター / 備品 の3タブ

**在庫数の表示ルール**
- 通常: 黒字で表示
- 基準在庫を下回る場合: 赤字で表示
- 発注済みがある場合: 「発注済: ○個」バッジを表示

**在庫数の変更操作**
| 操作 | 方法 |
|------|------|
| 1増やす | ＋ボタンをタップ |
| 1減らす | −ボタンをタップ（0以下にはならない） |
| 直接指定 | 在庫数をタップ → 数値入力ダイアログ |

**保存**
- ＋/−/直接入力で変更が生じると画面下部に「保存」ボタンが出現
- 「保存」タップ → 確認ダイアログ → Firestoreへ保存
- 変更件数・内容が確認ダイアログに表示される
- 保存と同時に履歴にニックネームつきで記録される

**基準在庫の設定**
- アイテム行を長押し → 「基準在庫: ○○」ダイアログ → 数値入力 → 保存
- 基準在庫は0以上の整数

**納品処理**
- 「発注済: ○個」バッジのあるアイテムに「納品」ボタンが表示される
- 「納品する」をタップ → 確認ダイアログ → 発注数が在庫数に加算 → 発注済みがクリアされる

**検索**
- 上部検索バーでコードまたは名前を絞り込み

**再読み込み**
- 右上の更新アイコンで最新データを取得

---

#### 【5】全店舗在庫確認

メニュー→「全店舗在庫確認」

- 商品 / テスター / 備品 の3タブ
- 縦軸: アイテム名、横軸: 店舗名のテーブル形式
- 基準在庫を下回るセルは赤字
- 読み取り専用（変更不可）

---

#### 【6】商品マスタ管理

メニュー→「商品マスタ管理」

**タブ**
- 商品 / テスター / 備品

**アイテムの追加**
1. 右下「＋」ボタン
2. コード（任意）・名前（必須）を入力
3. 「保存」

**アイテムの編集**
- 行をタップ → 編集ダイアログ → コード・名前を変更 → 「保存」

**アイテムの削除**
- 行を長押し（またはスワイプ）→ 削除確認 → 「削除」

**販売終了（廃番）設定**
- 編集ダイアログで「販売終了」をON
- 販売終了アイテムは発注リストから自動除外、在庫一覧でグレー表示

**並び順**
- コードのナチュラルソート順（例: T1 < T2 < T10）
- コードなしの場合は名前順

**検索**
- 上部検索バーで絞り込み

---

#### 【7】発注リスト

メニュー→「発注リスト」（※広告表示後に開く場合あり）

**表示条件**
- 基準在庫が設定されていて、かつ現在在庫が基準在庫を下回っているアイテムのみ表示
- 販売終了アイテムは除外

**表示内容**
- 店舗名・アイテム名・カテゴリ
- 現在庫 / 基準在庫 / 不足数
- 発注数量（変更可能、デフォルトは不足数）
- 発注済みバッジ（発注済みの場合）

**絞り込み**
- 上部チェックボックスで「商品 / テスター / 備品」を選択

**発注操作**
1. 発注数量を確認・変更
2. 「発注」ボタン
3. 確認ダイアログ「発注する」

**PDF出力**
- 右上の印刷アイコン → A4サイズのPDF生成 → 印刷または保存

---

#### 【8】特別発注・新規発注

メニュー→「特別発注・新規発注」

**対象**
通常の発注リスト（基準在庫管理）とは別に管理する特別品・新規品。

**発注種別**
- 特別発注 / 新規発注 / その他

**アイテム追加**
1. 右下「＋」ボタン
2. アイテム名・種別・メモ（任意）を入力
3. 「追加」

**仮発注**
- 各アイテム行の店舗欄に数量を入力 → 「仮発注する」確認ダイアログ → 確定

**納品処理**
- 「納品する」ボタン → 確認 → 納品済みとしてマーク

**アイテム削除**
- 削除ボタンでアイテムを削除

---

#### 【9】修正・追加履歴

メニュー→「修正・追加履歴」

- 直近100件を新しい順で表示
- 右上更新アイコンで最新取得

**各履歴の表示内容**
| 項目 | 内容 |
|------|------|
| 日時 | 変更された日時 |
| 店舗名 | 変更が行われた店舗 |
| アイテム名 | 変更されたアイテム |
| 種別 | 商品 / テスター / 備品 |
| 変更前 → 変更後 | 変更前後の在庫数 |
| 記録者 | 変更したユーザーのニックネーム |

---

#### 【10】店舗の並び替え

メニュー→「店舗の並び替え」

1. 各行右端の「≡」アイコンをドラッグして順番を変更
2. 「保存」で確定

---

#### 【11】組織管理（管理者専用）

メニュー→「組織管理」

**組織名の変更**
- 「組織名」欄を編集して保存

**ロゴのアップロード**
1. 「ロゴを変更」
2. 画像ファイル選択（JPEG/PNG推奨、幅300px・quality55でリサイズ保存）
3. トップバーにロゴが表示される

**招待コードの管理**
- 現在の招待コードを確認・コピー
- 「招待コードを変更」で新しいコードを設定（英数字・アンダースコアのみ）
- 変更すると古いコードは無効になる

**メンバー一覧**
- 管理者→メンバーの順でソート表示
- 表示内容: ニックネーム・メールアドレス・役職
- メンバー削除: 行右端のアイコン → 確認 → 削除（取り消し不可）

**組織の削除**
- 「組織を削除」で組織全体を削除（取り消し不可）

**組織を脱退（メンバー専用）**
- メニュー→「組織を脱退」→ 確認 → 脱退
- 脱退後は新しい組織作成または別の組織に参加可能

---

#### 【12】広告スペース管理（管理者専用）

メニュー→「広告スペース管理」

**広告モード**
| モード | 説明 |
|--------|------|
| 非表示 | 広告を一切表示しない |
| 自社メッセージ | 自組織の広告のみ表示 |
| 配信広告受信 | 他組織の広告も含めて表示 |

**広告スロット設定**
- 最大3スロット（通常）まで登録可能
- 各スロットに画像・メッセージ・URL（タップ時の遷移先）を設定

**広告の表示タイミング**
- 店舗一覧下部のバナー: 1分ごとにローテーション
- 発注リストを開く直前: 3秒間の全画面広告（3秒後に自動クローズ）

### よくある質問（FAQとして必ず含めること）

**Q. 在庫を変更したのに保存されていない**
A. ＋/−ボタン等で変更後、必ず画面下部の「保存」ボタンをタップしてください。タップせずに画面を離れると変更は破棄されます。

**Q. 発注リストにアイテムが表示されない**
A. 「基準在庫が設定されていて、かつ現在在庫が基準在庫を下回っている」アイテムのみ表示されます。また「販売終了」設定のアイテムは除外されます。

**Q. 新しい組織を作ったがログイン後に使えない**
A. 新規組織は統括管理者の承認が必要です。承認されるまで「承認待ち」画面が表示されます。承認後はアプリを再起動してください。

**Q. 招待コードがわからない**
A. 管理者が「組織管理」→「招待コード」から確認・共有できます。メンバーは管理者にお問い合わせください。

**Q. パスワードを忘れた**
A. ログイン画面「パスワードをお忘れですか？」からメールアドレスを入力するとリセットメールが届きます。

**Q. 履歴に記録者名が表示されない**
A. ニックネーム機能追加前の変更は記録者名が空白になる場合があります。以降の変更は正しく記録されます。

**Q. 広告を非表示にしたい**
A. 管理者が「広告スペース管理」→「非表示」モードを選択することで広告を無効化できます。

**Q. 店舗数・ユーザー数の上限を増やしたい**
A. 上限変更は統括管理者のみが行えます。「フィードバック→機能追加依頼」またはinfo@happy-bluebird.co.jp までご連絡ください。
